# SPOT-5 Benchmark（卫星拍摄调度基准）

## 问题描述

SPOT-5 卫星（2002-2015年运行）拍摄调度的经典约束优化问题，源自法国 ROADEF 2003 挑战赛和 CNES 的实际运营。

### 卫星配置

SPOT-5 携带三台成像仪器：
- 两台 HRG（高分辨率几何）相机
- 一台 HRS（高分辨率立体）相机

### 优化目标

在满足以下约束的同时，选择照片以最大化总利润：

1. **相机约束**：单图像使用一台相机（HRG 前、中或后）；立体图像需要同时使用两台 HRG 相机
2. **非重叠约束**：由于镜面转动时间限制，照片不能冲突
3. **数据流约束**：即时遥测带宽限制禁止某些组合
4. **存储约束**：星载记录容量限制所选图像总数（多轨道实例）

## 历史背景

| 年份 | 事件 |
|-----|------|
| 2002 | SPOT-5 卫星由 CNES 发射 |
| 2003 | ROADEF 2003 挑战赛发布原始遥测数据 |
| 2001-2003 | Vasquez & Hao 将物理约束抽象为冲突图（VCSP 公式） |
| 2021 | Wei & Hao 进一步简化并发布 21 个基准实例 |

## 数据格式

### .spot 文件格式

```
<总变量数>
<变量规格行...>
<总约束数>
<约束规格行...>
[<容量>]  # 仅适用于多轨道实例
```

### 变量规格

每个变量代表一个拍摄请求：
```
<var_id> <利润> <域大小> {<value_id> <记录器消耗>}* <域大小> [额外字段...]
```

### 约束规格

```
<arity> <var_id_1> ... <var_id_arity> {<forbidden_tuple>}*
```

- **arity**：变量数量（2 = 二进制，3 = 三元）
- **forbidden_tuple**：禁止的值组合

## 评估指标

- `profit` - 所选照片的总利润
- `weight` - 内存使用量（多轨道实例）
- `is_valid` - 是否满足所有约束

## 约束验证

### 二进制约束
如果两个变量都被选中，则它们的对值不能在禁止列表中。

### 三元约束
如果三个变量都被选中，则它们的三元组不能在禁止列表中。

### 内存容量约束（多轨道）
```
total_weight ≤ 200
```

## 实例类型

- **单轨道实例**（8, 54, 404, 408, 412, 5, 11, 28, 29, 42, 503, 505, 507, 509）：所有记录器消耗为 0
- **多轨道实例**（1401, 1403, 1405, 1502, 1504, 1506, 1021）：需要满足内存容量约束
