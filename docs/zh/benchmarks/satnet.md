# SatNet Benchmark（深空网络调度基准）

## 问题描述

深空网络（DSN）调度问题的强化学习基准。涉及最优分配地面站天线时间以与太阳系各地的航天器通信，同时遵守严格的物理和操作约束。

## 问题背景

深空网络是 NASA/JPL 的国际巨型无线电天线阵列，支持行星际航天器任务。调度问题需要在 1 周期内最大化成功通信轨道总时长，同时遵守：

- **可见周期（VP）约束**：只有当卫星与地面站有视线时才能通信（由轨道力学决定）
- **设置/拆卸要求**：每次轨道需要校准时间
- **非重叠约束**：天线不能同时处理多个传输
- **维护计划**：天线有计划停机时间

## 历史背景

| 年份 | 事件 |
|-----|------|
| 1963 | NASA 建立深空网络 |
| 2000s | DSN 操作自动调度系统开发 |
| 2021 | Chien 等人发布卫星调度 RL 基线（IEEE 航空航天会议） |
| 2021 | SatNet 基准数据集发布 |

## 决策变量

对于每个通信请求，决定：
- **是否**调度（可能部分满足或未满足）
- **使用哪个**天线（来自请求的兼容天线）
- **何时**调度（在有效可见周期内）
- **分配多长时间**（在 `duration_min` 和 `duration` 之间）

## 约束

1. **可见周期约束**：每个轨道必须完全在可见周期内
2. **非重叠约束**：同一天线上的轨道不能重叠
3. **设置/拆卸约束**：时间一致性
4. **维护约束**：不能与天线停机重叠
5. **最小持续时间约束**：每个轨道必须满足最小持续时间

## 目标

最大化总通信时长：
```
maximize: Σ (track.tracking_off - track.tracking_on) / 3600
```

## 数据格式

### problems.json

JSON 对象，week 键映射到请求数组：

```json
{
  "W10_2018": [
    {
      "subject": 521,
      "duration": 1.0,
      "duration_min": 1.0,
      "resources": [["DSS-34"], ["DSS-36"]],
      "setup_time": 60,
      "teardown_time": 15,
      "time_window_start": 1520286007,
      "time_window_end": 1520471551
    }
  ]
}
```

## 可用数据

5 周的 2018 年数据（W10, W20, W30, W40, W50），共 1,452 个请求。

## 评估指标

- `total_hours` - 总通信时长
- `request_coverage` - 请求覆盖率
- `antenna_utilization` - 天线利用率
